# Contributing to HKUST(GZ) Thesis Template

感谢你对本模板的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果你发现了 Bug 或有改进建议：

1. 先搜索 [Issues](https://github.com/crimson-and-clover/hkustgz-thesis-template/issues) 确保问题没有被报告过
2. 如果没有，创建一个新的 Issue，选择合适的模板：
   - **Bug Report**: 报告错误或问题
   - **Feature Request**: 建议新功能

### 提交代码

1. **Fork 仓库** 到你自己的账号
2. **创建分支** (`git checkout -b feature/AmazingFeature`)
3. **提交更改** (`git commit -m 'Add some AmazingFeature'`)
4. **推送分支** (`git push origin feature/AmazingFeature`)
5. **创建 Pull Request**

### 开发建议

- 保持代码风格一致
- 为复杂功能添加注释
- 更新相关文档（README 等）
- 确保更改不会破坏向后兼容性

## 代码规范

### LaTeX 代码

- 使用有意义的宏名称
- 添加注释说明复杂逻辑
- 遵循现有的缩进风格

### 提交信息

- 使用清晰的提交信息
- 简要描述改动内容
- 关联相关的 Issue 编号

## 审查流程

所有 Pull Request 都需要：

1. 通过 GitHub Actions CI 检查
2. 维护者审查
3. 解决所有审查意见

## 问题咨询

如有疑问，欢迎通过以下方式联系：

- 创建 [Discussion](https://github.com/crimson-and-clover/hkustgz-thesis-template/discussions)（如果启用）
- 发送邮件给维护者

## 许可证

通过贡献代码，你同意你的贡献将采用与原项目相同的 [MIT 许可证](LICENSE)。
